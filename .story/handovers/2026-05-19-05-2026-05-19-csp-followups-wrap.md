## Contract reference for future migrations

The dispatcher arg-passing convention (defined in commit `8b184fd`, documented in `static/js/actions.js` header):

```html
<!-- single arg -->
<button data-click="fn" data-foo="{{ x }}">…</button>

<!-- multi arg -->
<button data-click="addModule"
        data-type-id="{{ r.type_id }}"
        data-type-name="{{ r.type_name }}"
        data-slot-type="{{ r.slot_type }}">Add</button>

<!-- stopProp + arg -->
<button data-click="viewLedger" data-char-ids="{{ id }}" data-stop>View</button>
```

```js
window.fn = function() {
    // dispatcher calls fn.call(el, e) — `this` is the matched element
    var foo = this.dataset.foo;
    // ...
};
```

Handler functions that need to support BOTH dispatcher and legacy positional callers (e.g. callers from JS string-concat that ISS-020 will eventually retire) can detect via `this.dataset.<key>` presence — see `showModuleInfo` in `app/templates/fitting_tool.html:914` for the pattern.
